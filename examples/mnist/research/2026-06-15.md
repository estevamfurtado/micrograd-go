# MNIST: 89% → 95%

**Date:** 2026-06-15
**Goal:** Raise test accuracy from 89% to 95%.

## Current setup
- Architecture: `NewMLP(784, []int{16, 10})` — single hidden layer, 16 ReLU units.
- Loss: softmax + cross-entropy (correct).
- Init: weights uniform in [-1, 1], bias random in [-1, 1], no fan-in scaling.
- Optimizer: plain SGD, lr 0.05 with linear decay, no momentum.
- Batch 32, 5 epochs, full 60k train set.
- L2 reg alpha = 1e-4.
- Result: train 88.9%, test 89.1%.

## Hypothesis
The model is **underfitting, not overfitting** — train accuracy (88.9%) ≈ test
accuracy (89.1%), so it can't fit even the training set past ~89%. This points
at model **capacity** and **optimization**, not generalization. The gap to 95%
is driven, in order of impact:

1. **Hidden layer too small (16 units).** A single 16-unit hidden layer caps a
   MNIST MLP around 88-91%. This is most of the gap.
2. **No fan-in weight init.** First layer sums 784 weights in [-1, 1], giving
   large pre-activations that saturate ReLU before training starts. Becomes the
   dominant blocker once the network is widened.
3. **Plain SGD + only 5 epochs.** Slow convergence; loss trace has plateaued
   (still bouncing 0.5-1.1 at step ~9300) rather than still descending.

Not the problem: loss function (correct), [0,1] normalization, L2 reg (tiny,
and we're underfitting so it isn't hurting), batch size 32.
